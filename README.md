# SSH 文件传输

一个原生 macOS 的双窗格 SFTP 文件传输工具。适合在本机目录和远程服务器之间浏览、上传、下载、拖拽文件，也可以同时打开多个本地或远程会话。

应用由 SwiftUI 编写，SFTP 连接基于 [Citadel](https://github.com/orlandos-nl/Citadel)。日常使用不需要 Python、miniconda 或其它运行时。

## 运行环境

- macOS 14 或更新版本
- 常规文件传输不需要额外依赖
- “挂载”功能需要本机已经安装 macFUSE 和 sshfs；没有安装时，应用会直接提示该功能不可用

## 使用方式

1. 打开应用后，从顶部“服务器”下拉框选择一台主机。
2. 点击“连接”。应用会读取 `~/.ssh/config` 中的配置，也支持在应用里手动添加服务器。
3. 左右两侧都可以打开本地目录或远程服务器标签页。
4. 选中文件后，可以用底部按钮传输，也可以直接拖拽到另一侧。

如果当前聚焦的是本地目录，顶部服务器下拉框会显示“本地目录”。这时选择服务器并点击“连接”，会在对侧新建一个远程会话。

## 主要功能

- 双窗格浏览，本地和远程标签页可以在左右两侧自由组合
- 多标签会话，远程标签页之间互不影响，各自保持连接和路径
- 上传、下载、本地复制、远程中转传输
- 文件夹递归传输，传输进度、速度、剩余时间和日志可查看
- Finder / 本地行 / 远程行之间的拖拽传输
- 新建文件夹、重命名、删除；本地删除会进入废纸篓
- 按名称、大小、修改时间排序，目录始终排在前面
- 当前目录过滤和递归搜索
- 本地常用位置菜单：主目录、iCloud 云盘、桌面、文档、下载和外接磁盘
- 远程项目属性查看
- 连接结果、传输结果以轻量提示显示

## 挂载功能

当左右两个活跃会话正好是“一边本地、一边已连接远程”时，右下角的“挂载”按钮会变为可用。点击后，应用会把当前远程目录挂载到当前本地目录。

为了避免误伤本地文件，挂载前会做这些检查：

- 本地路径必须已经存在，并且必须是文件夹
- 本地路径不能是符号链接
- 本地文件夹必须为空
- 本地路径不能已经是其它挂载点
- 挂载前会二次确认

“管理挂载...”只显示通过本应用创建的挂载记录。打开时会检查记录是否仍然有效：

- 有效挂载可以在终端打开、在 Finder 打开、复制本地路径或取消挂载
- 失效记录可以重新挂载或删除记录
- 如果本地路径当前被其它挂载占用，应用不会尝试取消它

取消挂载也会二次确认，并且只会取消本应用记录且来源匹配的 sshfs 挂载；不会删除本地目录。

## 安全说明

- 连接远程服务器时会校验 `~/.ssh/known_hosts`，支持明文和哈希条目
- 未知主机会显示指纹，确认信任后才写入 known_hosts
- 主机密钥不匹配会拒绝连接
- 私钥只在本机读取；带口令私钥的口令只在内存中用于解密，不会保存
- 手动添加的密码服务器会使用本机钥匙串保存密码

## 构建

开发环境需要 Swift 6.x / Xcode。

```bash
swift build
swift run
```

生成可双击的 app：

```bash
./make-app.sh
open "SSH 文件传输.app"
```

也可以用 Xcode 打开：

```bash
open Package.swift
```
